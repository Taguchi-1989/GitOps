## プロンプト

あなたはWebフロントエンドのデザインエキスパートです。

添付の `flowops-slides.html` は「FlowOps」という業務フロー管理ツールのPR用HTMLスライド（全12枚、シングルファイル）です。
`slide-outline.md` は骨子（5W2Hフレームワーク）です。

### 現状
- 全12枚のスライドが動作しており、キーボード(矢印/Space/Home/End)・ボタン・スワイプで操作可能
- デザインはブルー系グラデーション、Noto Sans JP、16:9基準
- タイトルスライド(1)と最終スライド(12)はダーク背景、中間スライドは白背景

### 確認済みの問題点・改善したい点

1. **パディング/余白の調整**: 一部スライド(特にSlide 8 Workflow, Slide 10 Security)でコンテンツが窮屈。paddingやフォントサイズの微調整が必要
2. **タイトルスライドの演出強化**: 現在シンプルすぎるので、背景パターンやアニメーションで印象を強くしたい
3. **コードブロックの見た目**: YAMLサンプル(Slide 3)のシンタックスハイライトをもう少しきれいにしたい
4. **レスポンシブ対応**: 現在 `min(96vw, 1280px)` x `min(90vh, 720px)` だが、もっと小さい画面やフルHDプロジェクタでの表示を最適化したい
5. **アニメーション**: フェードイン以外に、カード要素のスタガーアニメーション（順番に表示される）を入れたい
6. **印刷対応**: `@media print` でスライドを1ページ1枚で印刷できるようにしたい
7. **全体的なデザインの洗練**: プレゼン資料としてのクオリティを上げてほしい

### 制約
- シングルHTMLファイルを維持すること（外部CSS/JSファイルに分離しない）
- Google Fonts (Noto Sans JP) のCDN読み込みはOK
- 既存の12枚のスライド構成・コンテンツは変更しない（デザイン/レイアウト/アニメーションのみ改善）
- JavaScriptのスライドナビゲーションロジックは現在のものを維持

### お願い
上記の改善点を反映した、改善版の `flowops-slides.html` を出力してください。
特にプレゼンで映えるデザインクオリティを重視してください。

## プロジェクト概要（Geminiへの補足コンテキスト）

FlowOpsは:
- 業務フロー（YAML）をGitで正本管理するWebプラットフォーム
- Issue起票 → LLMがJSON Patch自動生成 → レビュー → マージ の改善サイクル
- Next.js 14 + SQLite + Prisma + simple-git + Zod + Mermaid.js
- 6つのLLMプロバイダー対応（OpenAI/Anthropic/Gemini/Groq/Ollama/Custom）
- 非エンジニアでもWebUIから業務フロー改善に参加できる
