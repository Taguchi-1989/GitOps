## 2. PDCA を「回す」運用メカニズム（本書の核心）

### 2.1 二重の PDCA

FlowOps は **製品自身が業務 PDCA ツール**である。改善活動も同じ仕組みでドッグフーディングする。

```
[製品の PDCA = 利用者の業務改善]          [開発の PDCA = 本リポジトリの改善]
 Plan  : 改善カード(Issue)を起票           Plan  : 本書 §3 バックログを四半期レビュー
 Do    : ブランチ作成 → AI提案 → 適用       Do    : スプリント(2週)で着手、PR
 Check : フローに反映して結果確認           Check : §4 メトリクスを週次計測 + CI ゲート
 Act   : 標準化(standardize)で正本化         Act   : レトロ → 本書更新 → README/CLAUDE.md へ反映
```

> **ドッグフーディング原則**: 本書 §3 の各改善項目は、可能な限り FlowOps 上の「改善カード(Issue)」として起票して回す。自社ツールで自社改善を回すこと自体が、業務活用の最良のデモになる。

### 2.2 ケイデンス（いつ回すか）

| 周期 | イベント | アウトプット |
|---|---|---|
| 週次 | Check ミーティング（15 分） | §4 メトリクス確認、赤信号項目のエスカレーション |
| 隔週 | スプリントレビュー / レトロ | Done 確定、新規改善点を §3 に追記 |
| 四半期 | バックログ棚卸し | 優先度再評価、KPI 目標の再設定、本書の改訂 |

### 2.3 役割（簡易 RACI）

| 役割 | 責務 |
|---|---|
| プロダクトオーナー | §3 の優先度決定、業務 KPI の合意 |
| テックリード | 技術的受入条件(DoD)・CI ゲートの維持 |
| 改善担当(持ち回り) | 週次 Check の集計、レトロのファシリテート |

### 2.4 Definition of Ready / Done（入口・出口）

**Ready（着手可能）**: ① 課題と期待効果が 1 行で言える ② 受入条件(KPI もしくは検証手順)がある ③ 影響ファイルが特定済み。
**Done（完了）**: ① CI 全グリーン ② テスト追加 or 検証ログ添付 ③ 必要なら README/CLAUDE.md/本書を更新 ④ 改善カードを Check→Act(標準化)まで遷移。

## 4. メトリクス（Check で計測する対象）

> 計測できないものは改善できない。Check フェーズは下表を機械的に集計する。集計元は監査ログ(`AuditLog`)・CI・Langfuse を基本とする。

### 4.1 業務 KPI（製品価値）

| KPI | 定義 | 目標(初期) | 集計元 |
|---|---|---|---|
| 改善カード起票数 | 期間内の Issue 作成数 | 増加トレンド | AuditLog `ISSUE_CREATE` |
| サイクルタイム | 起票→標準化(Act)までの中央値 | < 5 営業日 | Issue 状態遷移 |
| AI 提案採択率 | 適用された提案 / 生成された提案 | > 60% | Proposal 状態 |
| 差し戻し率 | Decision Card で revise/stop となった割合 | 監視(過大なら品質課題) | GateEvaluation / approve API |
| 標準化率 | merged のうち standardize まで到達した割合 | > 80% | Issue 状態 |

### 4.2 技術 KPI（健全性）

| KPI | 目標 | 集計元 |
|---|---|---|
| CI 成功率(main) | > 95% | GitHub Actions |
| テストカバレッジ(core) | 下限維持・低下させない | vitest coverage |
| E2E クリティカルパス | 常時グリーン | Playwright(B-1) |
| 構造検証 警告数 | 0（全 spec/flows） | validate-structure(B-2) |
| 監査欠損 | 主要操作で AuditLog 欠損 0 | 監査整合チェック |

### 4.3 コスト KPI

| KPI | 目標 | 集計元 |
|---|---|---|
| LLM トークン/月 | 予算内 | Langfuse / LLM client ログ |
| 提案 1 件あたりコスト | トレンド低下 | コスト ÷ 生成数 |
