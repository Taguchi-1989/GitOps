# AI リスクアセスメント & 影響評価 手順

> AIMS 管理文書（ISO/IEC 42001 6.1.2 / 6.1.4 / 8.2-8.4 / A.5）｜状態: `active`｜更新: 2026-06-20

## 1. トリガ（いつ実施）

| トリガ | リスク評価 | 影響評価 |
|---|---|---|
| 新AI機能/ロジックの `active` 化前 | 必須 | 中リスク以上で必須 |
| 計算式・前提・閾値の変更 | 必須(差分) | 高リスク要素なら必須 |
| 個人データ/安全に関わる工程 | 必須 | **必須** |
| インシデント・差し戻し多発 | 再評価 | 再評価 |

**影響評価を要する変更は安全レビューゲート通過まで `active` 化しない**（Rule 2）。

## 2. リスク評価手順 (6.1.2 / 8.2)

対象特定 → リスク源同定（hazard-identification 流用＋AI固有） → 分析(起こりやすさ×影響度) → 受容基準(§4)照合 → 対応(回避/低減/移転/受容)を管理策に落とす → 記録（`aims:risk` Issue＋AuditLog、`formula_version`/`assumption_id` 紐付け）。

**AI固有リスク源**: 誤提案の本番反映 / データの偏り・陳腐化 / ハルシネーション・過信 / 機微データのLLM送信 / プロバイダ依存 / コスト暴走。

## 3. 影響評価手順 (6.1.4 / 8.4 / A.5)

リスクが組織内に閉じない場合、人・社会への影響を評価。各カテゴリで「有無→深刻度→可逆性→緩和策」を `aims:impact` Issue に記録、評価時点の assumptions を紐付け。

| カテゴリ | 問い |
|---|---|
| 個人 (A.5.4) | 特定個人の権利・安全・機会に影響するか |
| 集団 (A.5.4) | 特定集団に不利益・偏りが生じるか |
| 社会 (A.5.5) | 法規制・安全規格・社会的信頼に影響するか |

## 4. リスク受容基準 (6.1)

| レベル | 基準 | 必要な管理策 |
|---|---|---|
| 低 | 開発内に閉じ可逆 | 通常レビュー＋AuditLog |
| 中 | 業務影響あり可逆 | 影響評価＋承認者レビュー |
| 高 | 人身安全・個人/集団への重大影響・法規制 | **影響評価必須＋安全レビューゲート＋二重承認** |
| 受容不可 | 緩和不能の重大リスク | `active` 化禁止。設計変更 or 中止(stop) |

## 5. 是正・不適合フロー (10.1)

`aims:nonconformity` 起票 → 一次対応（必要なら `active` から外す/差し戻し） → 原因分析（人/仕組み/データ） → 是正処置（管理策追加をProposal化） → 有効性確認（`checkResult=effective`） → 標準化（`spec/**` 正本化＋AuditLog）。

高リスク・受容不可判定は四半期マネジメントレビュー(9.3)で必ずレビュー。
